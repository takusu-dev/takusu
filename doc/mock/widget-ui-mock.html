ポイント

  • 大きい 4x2 は情報量を保ちつつ、ブランド色の脈動ドットとプログレスバーで進行中タスクを強調。
  • 「次のタスク」は最大 4 件まで表示。5 件以上は「他 N 件」のオーバーフロー行にまとめる。
  • 時刻チップ＋タイトル＋緊急度ドットのコンパクトな一行アイテム。
  • 緊急度ドット: 赤 = must-do (a < 0.25)、黄 = caution (a < 0.5)、緑 = calm。
  • 固定予定には brand バッジ「固定」を付けて区別。
  • 小さいサイズでは情報を絞り、サイズに応じた別レイアウトを使う。

小さいサイズのパターン

同じ内容を 4x1 / 2x2 / 2x1 でそれぞれ最適な情報量に絞って表示。

サイズ別表示ルール

  • 4x2: ホーム画面のメイン widget。進行中タスク＋次のタスク最大 4 件。
  • 4x1: 1 行で「進行中」または「次のタスク」のみ。時刻・タイトル・件数 chip。
  • 2x2: 縦型。進行中 or 次のタスク 1 件を大きく。残り件数は chip で右下に。
  • 2x1: 最小。ブランドアイコン＋ラベル＋タイトル＋時刻。余白は極力詰めない。

実装イメージ

  • レイアウト: mobile/modules/takusu-widget/android/src/main/res/layout/takusu_widget_4x2.xml / takusu_widget_4x1.xml / takusu_widget_2x2.xml / takusu_widget_2x1.xml
  • サイズ別に minWidth/minHeight / targetCellWidth/targetCellHeight を定義。
  • RemoteViews 内で動的に緊急度ドットの色を変える（setInt + setColorFilter）。
  • プログレスバーは ProgressBar または ImageView + clipRect で再現。
