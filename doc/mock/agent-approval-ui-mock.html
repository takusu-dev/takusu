ポイント

  • 操作は badge、対象はタイトル/参照だけ表示。種別接頭辞（task など）は省略。
  • 固定予定（start_at あり）の場合は「固定」ラベルを出し、時間帯を 〜 で表示。
  • 非固定タスクは「期限」と「所要時間」を表示。プランナーが自動スケジュール。
  • 更新の diff は変わった部分だけ打ち消し線。同じ日なら日付は一回。
  • 習慣は「時間帯」「繰り返し」「所要」で表示。
  • habit の steps は各 step の固定/非固定と、step 間の依存関係を表示。
  • 警告は実際の approval.warnings のみ表示。常時表示の汎用メッセージは不要。
  • 補完理由など自明な inferred fields は非表示。
  • 権限セクションは default で折りたたみ。ヘッダーに展開矢印と許可状況を表示。
  • すでに許可済みの権限は UI に出さない。未許可のものだけ表示する。
  • 権限は target:operation（例: task:create）から導出する。
  • 展開すると「すべて許可」ボタンが表示され、未許可の権限をまとめて ON/OFF できる。
  • 「すべて許可」は ON 時に brand 色で塗りつぶされ、許可中であることを強調する。
  • トグル ON の行は brand/error の背景で変化を示す。
  • 削除など破壊的な操作は赤枠で目立たせる。

この HTML はあくまで mock です。実装時は mobile/src/components/ApprovalPanel.tsx の approval 部分を、PermissionsEditor の概念と組み合わせて置き換えてください。
