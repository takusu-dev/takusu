ポイント

  • 進捗入力は差分と累積を横並び: シート内に「差分」と「累積」の 2 つの入力欄を横に並べ、どちらからでも入力できます。その下に「全体」も入力できます。
  • 数量はタスク作成時に設定: quantity_total / quantity_unit / quantity_done は TaskAdd / TaskEdit で設定します。進捗記録や分割はこれらが設定されているタスクで使えます。
  • アプリ実装では絵文字を使わない: mock では記号を使っていますが、実際のアプリでは SVG / アイコンライブラリに置き換えてください。
  • Home view の FAB:
    • in_progress タスクがない → 「次にやるべきタスクを開始」ボタン（▶）
    • in_progress タスクがある → 「一時停止」ボタン（⏸）に変化
    • in_progress 時、FAB を軽く上にスライドすると「進捗記録」シートが開く
  • TaskDetailView (in_progress): status のすぐ下に「進捗」セクションを置き、ボタンは上から「pause / done」「split」「-1 / 記録 / +1」の 3 段に配置します。
  • 一時停止は進捗記録付き: pause ボタンを押すと進捗シートを開き、数量を入力してから pauseTaskWork を実行します。
  • 完了: done ボタンで completeTaskWork を呼び、作業セッションを閉じて実績時間を記録します。
  • 分割: modal 内で「全体」と「元タスクに残す数量」を確認しながら、残りを新タスクとして切り出します。

実装時は mobile/src/views/HomeView.tsx の FAB と mobile/src/views/TaskDetailView.tsx の「進捗」セクションを更新してください。
