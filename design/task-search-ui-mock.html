task search + completion UI mock (#921)

mobile / CLI / agent 共通のタスク検索構文と、入力補完のモックです。OR / AND / NOT、日程、スケジュール時間も検索対象に入れています。

論理演算（案）

  • 空白 — 暗黙の AND
  • OR — どちらかを満たす
  • NOT または先頭の - — 否定
  • ( ) — グルーピング
  • AND は明示的にも書ける（空白と同義）

検索 qualifier（案）

  • status:<pending|scheduled|in_progress|completed|skipped|overdue> — ステータス
  • title:<text> — タイトルの部分一致
  • desc:<text> / description:<text> — 説明の部分一致
  • habit:<h1> — 習慣参照
  • is:fixed, is:parallelizable, is:allows_parallel, is:overdue
  • has:description, has:completed_at, has:schedule
  • 自由語 — title / description の両方を部分一致

日付・時間 qualifier（案）

  • start:<op><date> — タスクの start_at
  • end:<op><date> — タスクの end_at
  • scheduled-start:<op><date> — スケジュール上の開始時刻
  • scheduled-end:<op><date> — スケジュール上の終了時刻
  • from:<date> / until:<date> — 従来の task 締切/開始範囲（互換）
  • <op> は >= <= > < =
  • .. で範囲: start:2026-07-25..2026-07-28
  • 日付のみ指定時はその日全体を含む（<= は 23:59:59、>= は 00:00）

completion ルール（案）

  • 入力中の最後の token を判定
  • st → status:、stat まででも補完
  • status: → status:pending, status:completed ...
  • habit: → h1, h2 ...
  • o → OR, n → NOT など
  • 自由語では既存タスクのタイトルから部分一致した候補を表示

この mock での制限

  • 日付は簡易的に文字列比較でフィルタ
  • 自由語は大文字小文字無視の部分一致
  • 並び順は「pending → 時系列」の簡易版
