ポイント

  • ツール request_user_input / correct_asr は複数の質問を配列で返す。
  • 自明な誤認識は LLM が推測して進み、固有名詞・同音異義語・文脈に依存する語だけを確認に回す。
  • 各質問には元の認識テキスト text と判断材料 for を表示する。
  • 入力欄は元のテキストで事前埋めしており、ユーザーは必要な部分だけ修正する。
  • 「そのまま続ける」は未修正のテキストで回答するのと同じ。
  • 確定後、各項目の text に修正後の文字列が入り、LLM にツール結果として返る。

ツール入出力イメージ

in:
[
  {
    "text": "さんた",
    "for": "人名・地名・固有名詞の可能性があるため、正しい表記を確認"
  },
  {
    "text": "やくそく",
    "for": "同音異義語が複数あるため、文脈から判断できない"
  }
]

out:
[
  { "text": "三田" },
  { "text": "約束" }
]
