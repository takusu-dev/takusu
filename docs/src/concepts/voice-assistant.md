# 音声アシスタント

takusu の **音声アシスタント** は、マイク入力を受け取り、STT でテキスト化、LLM で意図を解釈してタスク・スケジュール操作を行い、TTS で応答する仕組みです。

## STT（音声認識）

**Speech-to-Text** では、主に **Sherpa-ONNX** ローカル推論を使います。

- 録音: マイク入力を取得し、WAV 形式で処理
- フォーマット: f32 / i16 モノラル、16kHz リサンプリング
- モデル: `sherpa-sense-voice-int8`（初回自動ダウンロード）
- 言語: `auto`, `ja`, `en`, `zh`, `ko` 等

録音からテキスト化まではモバイルアプリ内で行い、テキスト化された内容のみをサーバーに送信します。

## TTS（音声合成）

**Text-to-Speech** では、以下のバックエンドを使えます。

- **Cartesia Sonic**: クラウド TTS、高品質・多言語対応
- **Android TTS**: モバイルアプリ内蔵の TextToSpeech エンジン

`takusu-audio` クレートは `TextToSpeech` トレイトを定義しており、新しいバックエンドを追加できます。

## Resident Agent

モバイルアプリでは、常駐型の **Resident Agent Button** をホーム画面に表示しています。

- タップ: Agent 画面（`/agent`）を開く
- 上スライド: タスク追加画面（`/task/add`）を開く
- 長押しによる音声入力: **未実装**

処理中・読み上げ中はアイコン・アニメーションで状態を表示します。

## 承認フロー

タスク作成・更新・削除などの永続的な変更は、LLM が提案した後にユーザーが承認するまで確定しません。これにより、音声入力の誤認識による意図しない変更を防ぎます。

## 連続音声セッション

ユーザーが明示的に開始する間は、以下を繰り返します。

```text
録音 → 認識 → 思考/ツール実行 → 応答（テキスト・TTS）
```

VAD による自動発話区間検出や TTS 中の barge-in は **未実装** です。録音の開始・終了はユーザーが手動で行います。

## プライバシー

- 常時リスニングは行わない
- マイク使用中はユーザーに通知を表示
- 対象外の音声は外部送信・永続化しない
- raw audio、transcript、機密情報をログに記録しない
