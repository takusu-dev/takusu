# タスク

takusu における **タスク** は、タイトルや説明に加え、スケジューリングに必要な数値情報を持ちます。

## 主要な属性

| 属性 | 意味 |
|------|------|
| `start` | 開始可能時刻（これより前に始められない） |
| `end` | 締切 |
| `avg_minutes` | 所要時間の平均（分） |
| `sigma_minutes` | 所要時間の標準偏差（分） |
| `depends` | 依存タスク ID 一覧 |
| `parallelizable` | 他のタスクと同時にできるか |
| `allows_parallel` | 実行中に他タスクを並列実行できるか |
| `abandonability` | 締切を違反した場合の許容度（0.0 〜 1.0） |
| `fixed` | 手動で固定されたタスクかどうか |

## 所要時間の見積り

`takusu-core` は所要時間を **正規分布** `NormalDist(mean=avg, sigma=sigma)` で表現します。

- `sigma` が大きいタスクは「不確実なタスク」として扱われ、スケジュール後方に余裕を持って配置されます。
- `sigma = 0` のタスクは時間が確定している予定と同等に扱えます。

## 並列性

- `parallelizable = true`: このタスクは他のタスクと同時に実行できます（例: スマホで調べ物）。
- `allows_parallel = true`: このタスク実行中に他のタスクを並列実行できます（例: 移動中）。
- 両方が `true` の場合のみ、重なりが許可されます。

## 依存関係

`depends` に指定したタスクは、実行が完了してからでないと始まりません。Solver は依存を満たす配置を優先します。

## abandonability（諦めやすさ）

`abandonability` は 0.0 〜 1.0 の値です。

- `0.0`: 締切を絶対に守るべき。
- `1.0`: 締切を違反しても許容する。
- タスクは **ドロップされません**。不可能な場合はスケジュール末尾に置かれ、ペナルティとして表現されます。

## 状態（status）

| 状態 | 意味 |
|------|------|
| `pending` | 未スケジュール |
| `scheduled` | スケジュール済み |
| `in_progress` | 作業中 |
| `completed` | 完了 |
| `skipped` | スキップ |

`schedule generate` には `pending` と `scheduled` のタスクのみが含まれます。
